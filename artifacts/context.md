# AI Native 组织学习路径设计 — 最终知识文档（物理宪法）

## 1. 领域核心挑战（一句话）

如何将 AI Native 组织所需的隐性能力（AI 思维、人机协同、工具链驾驭）转化为 90 天内可习得、可评估、可迭代的显性学习旅程。

## 2. 核心业务变量（最终版）

| 变量 | 公式 | 合格阈值 | 说明 |
|------|------|---------|------|
| 学习完成率 LCR | 已完成单元/总单元×100% | ≥ 80% | 路径可行性核心指标 |
| 能力提升度 CGI | (Post-Pre)/Pre×100% | ≥ 50% | 前后测对比 |
| 时间到生产力 TTP | 首次独立完成任务天数 | ≤ 60 天 | 加速效果验证 |
| 学习参与度 ES | 活跃时长/计划时长×互动归一化 | ≥ 0.7 | 学习动力指标 |
| 知识留存率 KRR | 30天延迟测试/即时测试×100% | ≥ 50% | 间隔重复效果 |
| AI 工具使用频率 ATF | 日均 AI 工具调用次数 | ≥ 5 次/天 | AI 思维养成标志 |
| 知识分享活跃度 KSA | 每人每月分享次数 | ≥ 1 次/月 | 从学习者到贡献者转变 |
| 新人满意度 NPS | 推荐者% - 贬损者% | ≥ 40 | 体验质量综合指标 |

## 3. 验收的金标准

### 3.1 学习路径设计验收
- **路径完整性**：覆盖 30-60-90 三阶段，每阶段有明确目标、任务、里程碑
- **时长合理性**：每天学习时长 1-2 小时，每周实践环节 ≥ 2 个
- **认知递进性**：符合布鲁姆分类学（记忆→理解→应用→分析→评价→创造）
- **评估节点**：每阶段末有标准化评估，包含知识测试 + 实操考核 + 同伴评审

### 3.2 Demo 验收
- **可运行**：部署至公网，提供可访问链接
- **功能完整**：支持路径展示、进度追踪、评估反馈
- **用户体验**：界面美观、交互流畅、移动端适配

## 5. 边界条件处理

| 边界情况 | 预期行为 | 验证方法 |
|---------|---------|---------|
| 学员无 AI 基础 | 提供"零基础"预备模块 | 检查是否有 AI 101 内容 |
| 学员已有丰富经验 | 提供"跳级测试"，允许加速 | 检查是否有前置评估 |
| 学习中断（请假等） | 支持断点续学 | 检查进度持久化 |
| 移动端访问 | 响应式设计 | 检查移动端适配 |
| 多角色（学员/导师/管理员） | 权限隔离 | 检查角色系统 |
| 评估失败 | 提供补考机制和错题回顾 | 检查重试逻辑 |

## 6. 新手避坑指南

1. **内容过载**：每天 ≤ 2 小时学习，每个模块 ≤ 15 分钟
2. **理论脱节**：每个知识点配实战练习（"看完即练"模式）
3. **一刀切路径**：提供角色分支（美术/策划/程序/运营）
4. **缺乏即时反馈**：AI 实时评估 + 导师周评
5. **忽视遗忘曲线**：7 天/14 天/30 天间隔复习提醒

## 7. MVP 验证方案

1. **5 分钟验证**：
   - 打开 Demo 首页，确认 30-60-90 三阶段路径可视化正常渲染
   - 点击任意学习单元，确认内容详情可查看
   - 切换角色（美术/策划/程序），确认路径动态调整
   - 查看进度仪表盘，确认数据展示正常
2. **通过标准**：路径完整、交互流畅、数据可视化正确
