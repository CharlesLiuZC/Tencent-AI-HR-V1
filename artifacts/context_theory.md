# AI Native 组织学习路径设计 — 核心理论速查卡

> **该领域的根本挑战**：如何将"AI 思维方式"这一隐性知识体系，转化为可量化、可分阶段习得的显性学习路径，使新人在 90 天内完成从"AI 工具使用者"到"AI 协作共创者"的蜕变。

---

## 1. 领域定义与核心术语

### 领域定义
AI Native 组织学习路径设计属于**组织发展（OD）与人才发展（TD）的交叉领域**，其本质是将 AI 时代的能力模型拆解为可习得的技能单元，并按照认知负荷理论编排为递进式学习旅程。

### 核心术语（8 个）

| 术语 | 一句话解释 |
|------|-----------|
| **AI Native** | 以 AI 为核心生产力的组织形态，AI 不是工具而是协作伙伴 |
| **学习路径（Learning Path）** | 按照认知递进原则编排的有序学习活动序列 |
| **认知学徒制（Cognitive Apprenticeship）** | 通过示范、指导、脚手架逐步培养专家思维的教学模式 |
| **TPACK 框架** | 技术知识（TK）、教学法知识（PK）、内容知识（CK）的交叉整合模型 |
| **Zone of Proximal Development (ZPD)** | 维果茨基的最近发展区，在 AI 辅导下能达到但独立无法达到的能力区间 |
| **Micro-learning** | 将学习内容拆解为 5-15 分钟的碎片化学习单元 |
| **Flipped Classroom** | 翻转课堂，自学理论 + 课堂实践的教学范式 |
| **Competency Model** | 能力模型，定义岗位所需的知识、技能和行为标准 |

## 3. 经典算法与解决方案流派

| 流派 | 核心思路 | 优势 | 劣势 | 适用场景 |
|------|---------|------|------|---------|
| ADDIE 模型 | 分析→设计→开发→实施→评估 | 结构清晰 | 周期长 | 大型企业标准化培训 |
| SAM 迭代模型 | 小步快跑敏捷开发 | 快速迭代 | 需高频协作 | 快速变化的技术领域 |
| Kirkpatrick 四层评估 | 反应→学习→行为→结果 | 评估全面 | 实施复杂 | 需证明培训价值 |
| 70-20-10 法则 | 70%实践+20%辅导+10%课堂 | 符合成人学习规律 | 需强导师体系 | AI Native 组织 |
| 设计思维 | 共情→定义→构思→原型→测试 | 用户中心 | 需跨职能协作 | 新型学习体验设计 |

### 核心公式

- **艾宾浩斯遗忘曲线**：`R = e^(-t/S)` — R=记忆保留率，t=时间，S=记忆强度
- **认知负荷理论**：`Total = Intrinsic + Extraneous + Germane`

## 4. 验收的金标准

| 指标 | 测量方法 | 合格阈值 |
|------|---------|---------|
| 学习路径完成率 | LMS 数据 | ≥ 80% |
| 能力评估通过率 | 标准化测试 | ≥ 75% 达到"熟练" |
| 新人满意度 NPS | 问卷 | NPS ≥ 40 |
| 独立任务完成时间 | 项目追踪 | ≤ 60天 |
| AI 工具使用频率 | 系统日志 | 日均 ≥ 5次 |
| 知识分享活跃度 | 平台统计 | 每人每月 ≥ 1次 |
| 导师满意度 | 导师反馈 | ≥ 4.0/5.0 |

## 5. 新手必踩的坑

1. **学习内容过载** → 放弃率高 → 控制每天 1-2 小时
2. **理论与实践脱节** → 遗忘快 → 每个知识点配实战练习
3. **一刀切路径** → 快的等慢的 → 提供可选的加速/减速路径
4. **忽视社交学习** → 动力不足 → 建立学习小组
5. **缺乏即时反馈** → 重复错误 → AI 实时反馈 + 导师 review
6. **评估方式单一** → 测过但不会用 → 多元评估
7. **忽视遗忘曲线** → 学完就忘 → 间隔重复
8. **缺乏个性化** → 学非所需 → 入门测评 + 自适应路径

## 6. 理论基础

- **布鲁姆认知分类学**：记忆→理解→应用→分析→评价→创造
  - 30天(记忆+理解) → 60天(应用+分析) → 90天(评价+创造)
- **库伯学习圈**：具体经验→反思观察→抽象概念化→主动实验
- **维果茨基 ZPD**：学习目标 = 当前能力 + AI 辅导下可达水平

## 7. MVP 验证方案（5 分钟）

1. 绘制 30-60-90 天学习路径可视化图
2. 计算关键指标（完成率、能力提升度）
3. 验证每天学习时长 ≤ 2 小时
4. 通过标准：路径完整、时长合理、评估节点明确
